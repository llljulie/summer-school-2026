# Вопросы заказчику — «Скалодром»

> 10 ключевых вопросов для проектирования клиентского мобильного приложения и API.
> **Ответы зафиксированы по итогам сессии elicitation с заказчиком (Оля).**
> **Обновление от 04.07.2026** — добавлены уточнения по прокату, отмене, уведомлениям, рейтингу и лояльности.

## 1. Границы первой версии (MVP)

**Вопрос:** Что обязательно должно работать к запуску сезона, а что можно отложить?

**Ответ заказчика:**


| Функция                                                    | MVP        |
| ---------------------------------------------------------- | ---------- |
| Просмотр слотов (7 дней) и запись на тренировку            | ✅          |
| Выбор «со своим / прокат» при записи                       | ✅          |
| Отмена записи клиентом                                     | ✅          |
| Push-уведомления (запись, напоминания, отмена скалодромом) | ✅          |
| Оценка инструкторов после тренировки                       | ✅          |
| Программа лояльности для постоянных клиентов               | ✅          |
| Онлайн-оплата при записи                                   | ❌ отложено |

## 2. Регистрация и вход клиента

**Вопрос:** Как клиент получает доступ к приложению?

**Ответ заказчика:** Самостоятельная регистрация по **телефону + SMS-код**.

---



## 3. Сценарий записи от начала до конца

**Вопрос:** Кого и как записываем?

**Ответ заказчика:** Запись **только на себя**. Путь в приложении:

1. Выбор слота
2. Выбор снаряжения (своё / прокат)
3. Подтверждение
4. Экран «Вы записаны»

Запись нескольких участников с одного аккаунта — **не в MVP**.

---



## 4. Прокат снаряжения при записи

**Вопрос:** Как работает прокат при записи?

**Ответ заказчика:** 

- Выбор «**своё / прокат**» — **обязателен** при каждой записи.
- При выборе проката **стоимость увеличивается автоматически** — клиент видит итоговую цену до подтверждения.
- Если в слоте **все комплекты проката заняты**, клиент может записаться **только со своим снаряжением** (кнопка «Прокат» становится неактивной с пояснением).

---



## 5. Правила отмены клиентом

**Вопрос:** По каким правилам клиент может отменить запись?

**Ответ заказчика:** 


| Период до начала тренировки   | Что происходит                                                                             |
| ----------------------------- | ------------------------------------------------------------------------------------------ |
| **> 10 минут**                | Отмена **разрешена без штрафа**. Место освобождается — его может занять другой клиент.     |
| **≤ 10 минут (включительно)** | Отмена **разрешена, но со штрафом**. Место освобождается — его может занять другой клиент. |


> **Важно:** место освобождается в **обоих** случаях. Разница только в наличии штрафа при отмене за 10 минут и менее.

> **Штрафная механика:** подлежит уточнению с командой разработки. Варианты: денежный штраф, списание баллов лояльности, штрафной балл (при накоплении 3 штрафных баллов — блокировка записи на сутки). Минимальная реализация в MVP — **штрафной балл** с возможностью доработки.

**Исходная боль из брифа:** «за 10 минут до начала — это уже проблема, место простаивает» — теперь решается через штраф, а не через запрет отмены.

---



## 6. Оплата в первой версии

**Вопрос:** Как устроена оплата в MVP?

**Ответ заказчика:** **Онлайн-оплаты нет.** После записи приложение **показывает цену** (включая прокат, если выбран); оплата — **наличными или переводом** на месте / по договорённости. Статусы «ожидает оплаты / оплачено» в приложении — **не требуются**.

---



## 7. Push-уведомления — минимальный набор

**Вопрос:** Какие уведомления обязательны в первой версии?

**Ответ заказчика:** 

- **Напоминания** о предстоящей тренировке — **за 24 часа и за 2 часа** — канал доставки **SMS**.
- **Подтверждение записи** — **push**.
- **Уведомление об отмене скалодромом** с причиной — **push**.
- **Приглашение оценить инструктора** после тренировки — **push** (спустя 1 час после окончания).

---



## 8. Оценка инструкторов

**Вопрос:** Нужна ли оценка инструкторов в MVP?

**Ответ заказчика:** 

- **Да.** Формат — **звёзды 1–5**, оценить можно **в течение 24 часов** после тренировки.
- **Средний рейтинг инструктора** (звёзды + количество оценок) **показывается клиенту до записи** — в карточке слота.

---



## 9. Платформы и условия использования

**Вопрос:** Какие платформы обязательны?

**Ответ заказчика:** На старте — **только Android**. iOS — **не в первой поставке**.

Офлайн-режим не требуется; достаточно работы **онлайн** (в том числе в зале).

---



## 10. API и интеграция с существующим бэкендом

**Вопрос:** Состояние API и интеграции с бэкендом?

**Ответ:** Вопрос **не задаётся заказчику** — выносится на **команду бэкенда / аналитика** (контракт API, готовность операций, тестовый стенд). Заказчик, как правило, не обладает техническим пониманием этой части.

**Для команды разработки (вне сессии с заказчиком):**

- согласовать контракт клиентского API;
- проверить готовность: слоты, бронирование, отмена, push/SMS-токены, регистрация по телефону, оценка инструкторов, программа лояльности;
- обеспечить тестовый стенд для разработки и приёмки.

---



## 11. Программа лояльности (добавлено)

**Вопрос:** Какая механика программы лояльности в MVP?

**Ответ заказчика:** 

- **Скидка 50% на каждое 10-е посещение**.
- Клиент видит в профиле свой **счётчик посещений** и статус («До скидки осталось X тренировок»).
- Скидка применяется **автоматически** при записи на 10-ю тренировку (цена уменьшается на 50% до подтверждения).

## Итоговое резюме для команды разработки



### Обновлённая логика ключевых сценариев


| Сценарий             | Поведение                                                                                     |
| -------------------- | --------------------------------------------------------------------------------------------- |
| **Прокат**           | Цена динамически пересчитывается; если свободных комплектов нет — принудительно только «своё» |
| **Отмена (>10 мин)** | Разрешена, без штрафа, место освобождается                                                    |
| **Отмена (≤10 мин)** | Разрешена, со штрафом, место освобождается                                                    |
| **Уведомления**      | Напоминания — SMS, всё остальное — push                                                       |
| **Рейтинг**          | Показывать в карточке слота до записи                                                         |
| **Лояльность**       | Скидка 50% на каждое 10-е посещение, счётчик в профиле                                        |

## Открытые вопросы для команды разработки (не к заказчику)

1. **Механика штрафа** при отмене ≤10 минут: денежный штраф / списание баллов / штрафные баллы с блокировкой — выбрать и реализовать на бэкенде.
2. **Прокатный фонд**: как бэкенд возвращает признак «есть ли свободные комплекты на этот слот» (поле `rental_available` или `rental_stock`)?
3. **Лояльность**: кто считает количество посещений — бэкенд (предпочтительно) или клиент? Согласовать API-метод для получения счётчика.
4. **Тестовый стенд**: сроки готовности для фронтенда.
