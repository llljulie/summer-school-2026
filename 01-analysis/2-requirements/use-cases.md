## UC-001 · P1 · Регистрация и вход по SMS

| Поле | Значение |
|------|----------|
| **ID** | UC-001 |
| **Название** | Регистрация и вход по SMS |
| **Актор** | Клиент |
| **Цель** | Получить доступ к приложению |
| **Предусловия** | Приложение установлено; есть сеть |
| **Постусловия (успех)** | Клиент авторизован |

### Основной поток

1. Клиент открывает приложение.
2. Клиент вводит номер телефона с использованием маски ввода (только цифры и символ `+`).
3. Система выполняет валидацию номера на клиенте.
4. При валидном номере система отправляет SMS-код через API.
5. Клиент вводит код.
6. Система проверяет код; при успехе создаёт сессию (новый или существующий аккаунт).
7. Клиент попадает на главный экран (расписание / мои записи).

### Альтернативные потоки

**2a. Неверный формат телефона** — система показывает сообщение: «Введите корректный номер телефона в формате +7 XXX XXX-XX-XX»; поток с шага 2.

**4a. Неверный или просроченный код** — система сообщает об ошибке; клиент может запросить код повторно с таймером 60 секунд (FR-001).

**4b. Превышен лимит попыток** — система показывает сообщение: «Слишком много попыток. Попробуйте через 15 минут» (FR-033).

**4c. Клиент уже зарегистрирован** — выполняется вход без создания дубликата аккаунта (FR-022).

**Связи:** FR-001, FR-022, FR-027, FR-033, US-001, US-012

## UC-002 · P1 · Просмотр расписания слотов

| Поле | Значение |
|------|----------|
| **ID** | UC-002 |
| **Название** | Просмотр расписания слотов |
| **Актор** | Клиент |
| **Цель** | Найти подходящую тренировку |
| **Предусловия** | Клиент авторизован |
| **Постусловия** | Клиент видит актуальный список слотов или empty state |

### Основной поток

1. Клиент открывает экран расписания.
2. Система запрашивает слоты на **7 дней** через API.
3. Система отображает список с атрибутами слота (FR-005), включая **средний рейтинг инструктора**, в **хронологическом порядке** (от ближайшего к самому отдаленному).
4. Клиент выбирает слот для записи → **UC-003**.

### Альтернативные потоки

**3a. Слотов нет** — отображается empty state «Пока нет доступных тренировок на выбранные даты. Попробуйте изменить период поиска» с кнопкой «Изменить даты» (FR-004).

**3b. Ошибка сети / API** — сообщение об ошибке и кнопка «Повторить» (NFR-008).

**4a. Фильтр по датам (P3)** — клиент задаёт период «с / по»; система запрашивает слоты за выбранный интервал (FR-003).

**Связи:** FR-002, FR-003, FR-004, FR-005, FR-019, US-003, US-013, BR-005, BR-007

## UC-003 · P1 · Запись на тренировку

| Поле | Значение |
|------|----------|
| **ID** | UC-003 |
| **Название** | Запись на тренировку |
| **Актор** | Клиент |
| **Цель** | Забронировать место в группе на себя |
| **Предусловия** | Клиент авторизован; слот отображается как доступный |
| **Постусловия (успех)** | Создана бронь; место зарезервировано на бэкенде |

### Основной поток

1. Клиент выбирает слот из расписания.
2. Система показывает детали слота и переход к выбору снаряжения.
3. Клиент выбирает **«Со своим»** или **«Нужен прокат»** (обязательный шаг).
4. Система показывает экран подтверждения с итоговой ценой (тренировка + прокат, если выбран).
5. Клиент подтверждает запись.
6. Система отправляет запрос бронирования в API (с Idempotency-Key) и ожидает ответ не более 5 секунд (FR-030).
7. При успехе — экран **«Вы записаны»** с деталями (FR-008, FR-013).
8. Система инициирует push-уведомление о подтверждении (FR-021).

### Альтернативные потоки

**3a. Нехватка прокатного фонда** — кнопка «Нужен прокат» неактивна с подсказкой «Весь прокат на это время занят. Выберите "Со своим снаряжением"» (FR-024).

**3b. Ошибка API проката** — оба варианта доступны, но при подтверждении запрос к API завершается ошибкой с сообщением «Не удалось проверить доступность проката. Попробуйте позже».

**5a. Клиент уже записан на этот слот** — API возвращает ошибку 409, приложение показывает: «Вы уже записаны на эту тренировку» (FR-006).

**6a. Превышен таймаут (5 секунд)** — показывается сообщение: «Сервер временно недоступен. Пожалуйста, повторите попытку» (FR-030).

**6b. Мест нет / слот закрыт** — API возвращает отказ; система показывает ошибку; бронь не создаётся (FR-009, BR-002).

**6c. Слот отменён скалодромом** — запись недоступна (FR-016).

**5b. Клиент отменяет на шаге подтверждения** — возврат к расписанию без создания брони.

**Связи:** FR-006, FR-007, FR-008, FR-009, FR-013, FR-021, FR-030, US-004, US-005, US-011, BR-001, BR-002

## UC-004 · P1 · Просмотр своих записей

| Поле | Значение |
|------|----------|
| **ID** | UC-004 |
| **Название** | Просмотр своих записей |
| **Актор** | Клиент |
| **Цель** | Контролировать предстоящие и прошлые брони |
| **Предусловия** | Клиент авторизован |
| **Постусловия** | Отображён актуальный список броней клиента |

### Основной поток

1. Клиент открывает раздел «Мои записи».
2. Система запрашивает брони клиента через API.
3. Система отображает список с датой, временем, форматом, инструктором, снаряжением, статусом и ценой.
4. Клиент может выбрать бронь для отмены → **UC-005** или оценки → **UC-008**.

### Альтернативные потоки

**3a. Нет активных записей** — empty state с предложением перейти к расписанию.

**3b. Бронь «Отменена скалодромом»** — отображаются статус и причина (FR-014); отмена клиентом недоступна.

**Связи:** FR-012, FR-014, US-002

## UC-005 · P1 · Отмена записи клиентом

| Поле | Значение |
|------|----------|
| **ID** | UC-005 |
| **Название** | Отмена записи клиентом |
| **Актор** | Клиент |
| **Цель** | Освободить место при изменении планов |
| **Предусловия** | Есть активная бронь |
| **Постусловия (успех)** | Бронь отменена; место освобождено на бэкенде |

### Основной поток

1. Клиент открывает «Мои записи» и выбирает предстоящую бронь.
2. Клиент нажимает «Отменить запись».
3. Система проверяет, сколько времени осталось до начала слота.
4. Если до начала **> 10 минут**, система запрашивает подтверждение.
5. Клиент подтверждает.
6. Система отправляет запрос отмены в API.
7. Статус брони обновляется; клиент видит подтверждение отмены.

### Альтернативные потоки

**3a. До начала **≤ 10 минут** — система показывает модальное окно с текстом: **«Вы отменяете запись за X минут до начала. Будет начислен штрафной балл. Продолжить?»** и кнопками «Да, отменить» и «Нет, вернуться». При подтверждении — отмена с начислением штрафного балла (FR-011).**

**3b. Бронь уже «Отменена скалодромом»** — отмена клиентом недоступна.

**3c. Клиент находится под блокировкой (3 штрафных балла)** — отмена существующей записи доступна (FR-010).

**6a. Ошибка API** — сообщение об ошибке; бронь остаётся активной до успешной отмены.

**Связи:** FR-010, FR-011, US-006

## UC-006 · P1 · Обработка отмены тренировки скалодромом

| Поле | Значение |
|------|----------|
| **ID** | UC-006 |
| **Название** | Обработка отмены тренировки скалодромом |
| **Актор** | Клиент (получатель); инициатор — администратор через существующую инфраструктуру |
| **Цель** | Быть проинформированным об отмене с причиной |
| **Предусловия** | У клиента была активная бронь на слот |
| **Постусловия** | Бронь в статусе «Отменена скалодромом»; клиент уведомлён |

### Основной поток

1. Администратор отменяет слот в существующей системе (вне скоупа клиентского приложения).
2. Бэкенд обновляет статус всех связанных броней и причину отмены.
3. Клиентское приложение получает обновление (push / polling при открытии).
4. Система отправляет **push-уведомление** с причиной (FR-015).
5. В «Мои записи» отображаются статус и причина (FR-014).
6. Слот исключается из доступных для новой записи (FR-016).

### Альтернативные потоки

**4a. Push недоступен** — при следующем открытии приложения клиент видит модальное окно с причиной отмены (FR-015).

**Связи:** FR-014, FR-015, FR-016, US-007, BR-009, R-008

## UC-007 · P2 · SMS-напоминание о тренировке

| Поле | Значение |
|------|----------|
| **ID** | UC-007 |
| **Название** | SMS-напоминание о тренировке |
| **Актор** | Клиент (получатель) |
| **Цель** | Не забыть о предстоящей тренировке |
| **Предусловия** | Активная бронь; до начала 24 ч или 2 ч |
| **Постусловия** | Клиент получил SMS |

### Основной поток

1. Бэкенд по расписанию определяет брони, требующие напоминания.
2. За **24 часа** до начала бэкенд отправляет SMS с деталями тренировки и адресом скалодрома.
3. За **2 часа** до начала бэкенд отправляет повторное SMS-напоминание.

### Альтернативные потоки

**2a. Бронь отменена** — напоминания не отправляются.

**2b. Бронь «Отменена скалодромом»** — напоминания не отправляются.

**Связи:** FR-017, FR-018, US-008, BR-006

## UC-008 · P2 · Оценка инструктора

| Поле | Значение |
|------|----------|
| **ID** | UC-008 |
| **Название** | Оценка инструктора |
| **Актор** | Клиент |
| **Цель** | Оценить качество проведённой тренировки |
| **Предусловия** | Тренировка завершена; прошло не более 24 ч; поле `attended = true` в данных брони |
| **Постусловия** | Оценка сохранена на бэкенде |

### Основной поток

1. Клиент открывает завершённую тренировку в «Мои записи» или разделе «История», или переходит по push-уведомлению с глубокой ссылкой.
2. Система проверяет окно 24 ч и поле `attended` в данных брони.
3. Клиент выбирает оценку **1–5 звёзд**.
4. Система отправляет оценку через API.
5. Система подтверждает сохранение. Кнопка оценки становится неактивной с текстом: «Вы уже оценили эту тренировку» (FR-019).

### Альтернативные потоки

**2a. Прошло более 24 ч** — оценка недоступна.

**2b. Оценка уже поставлена** — кнопка неактивна с текстом «Вы уже оценили эту тренировку».

**Связи:** FR-019, US-009, BR-007
