## FR-001 · P1 · Регистрация по телефону и SMS-коду

**Описание:** Клиент регистрируется самостоятельно: вводит номер телефона, получает SMS с кодом, подтверждает код и создаёт учётную запись.

**Входные данные:** номер телефона в международном формате E.164 (например, `+79001234567`).

**Валидация номера телефона:**

- При вводе номера телефона используется маска ввода: допускаются только цифры и символ `+`.
- Клиент может вводить номер как с `+`, так и без него — приложение приводит к единому формату E.164.
- Валидация выполняется на стороне клиента **перед** отправкой запроса на получение SMS-кода.
- Номер считается валидным, если:
  - содержит от 10 до 15 цифр (согласно стандарту E.164);
  - начинается с символа `+` и кода страны (например, `+7` для России) после приведения к формату.
- При невалидном номере показывается сообщение: **«Введите корректный номер телефона в формате +7 XXX XXX-XX-XX»**.

**Результат:** авторизованная сессия клиента.

**Отправка кода повторно:** При неудачной попытке получения или ввода кода клиент может запросить код повторно. Доступна кнопка **«Отправить повторно»** с таймером обратного отсчета **60 секунд**. Повторная отправка доступна только после истечения таймера.

**Связи:** US-001, UC-001

## FR-002 · P1 · Просмотр слотов на ближайшие 7 дней

**Описание:** По умолчанию приложение отображает список доступных тренировочных слотов на **7 календарных дней** от текущего момента.

**Сортировка:** Слоты отображаются в **хронологическом порядке** (от ближайшего к самому отдаленному).

**Источник данных:** API существующего бэкенда (read-only).

**Связи:** BR-005, US-003, UC-002

## FR-003 · P1 · Фильтр слотов по произвольному периоду дат

**Описание:** Клиент может изменить период просмотра расписания (дата «с» / «по») для горизонта **более 7 дней**.

**Связи:** BR-011, UC-002 (4a)

## FR-004 · P1 · Empty state при отсутствии слотов

**Описание:** Если в выбранном периоде нет доступных тренировок, отображается понятное сообщение: **«Пока нет доступных тренировок на выбранные даты. Попробуйте изменить период поиска»** и кнопка **«Изменить даты»**.

**Связи:** R-027, UC-002 (3a)

## FR-005 · P1 · Карточка слота с обязательными атрибутами

**Описание:** Каждый слот в списке содержит минимум:

- дата и время начала;
- длительность (~1,5 ч);
- формат / зона (новички / опытные, болдеринг / трассы);
- имя инструктора;
- **средний рейтинг инструктора** (по шкале 1–5) и **количество оценок**;
- количество **свободных мест** (с учётом лимита 8 для новичковых / 16 для остальных);
- цена тренировки;
- адрес скалодрома.

**Источник:** контракт API (R-015).

**Связи:** US-003, UC-002

## FR-006 · P1 · Запись на тренировку только на себя

**Описание:** Авторизованный клиент бронирует **одно место для себя** на выбранный слот. Запись нескольких участников с одного аккаунта — не поддерживается.

**Повторная запись на тот же слот:** Если клиент уже записан на слот и пытается записаться снова, бэкенд возвращает ошибку, приложение показывает сообщение: **«Вы уже записаны на эту тренировку»**.

**Поток:** слот → снаряжение → подтверждение → экран «Вы записаны».

**Связи:** BR-001, US-004, UC-003

## FR-007 · P1 · Обязательный выбор снаряжения при записи

**Описание:** На шаге записи клиент **обязан** указать один из вариантов:

- **Со своим** — скальники и страховочная система;
- **Нужен прокат** — аренда снаряжения скалодрома.

Если на слоте все комплекты проката заняты (API возвращает `rental_available = false`), кнопка «Нужен прокат» становится неактивной с подсказкой: **«Весь прокат на это время занят. Выберите "Со своим снаряжением"»**. При ошибке API проката оба варианта остаются доступными, но при подтверждении запрос к API завершается ошибкой с сообщением: **«Не удалось проверить доступность проката. Попробуйте позже»**.

**Связи:** US-005, UC-003 (3)

## FR-008 · P1 · Экран подтверждения успешной записи

**Описание:** После успешного бронирования клиент видит экран «Вы записаны» с ключевыми деталями: дата/время, формат, инструктор, выбор снаряжения, цена. При выборе проката отображается отдельная строка со стоимостью проката.

**Связи:** UC-003 (5)

## FR-009 · P1 · Обработка отказа бронирования при отсутствии мест

**Описание:** Если бэкенд возвращает отказ (мест нет / слот закрыт), приложение:

- не создаёт локальную «успешную» бронь;
- показывает понятное сообщение об ошибке;
- предлагает вернуться к списку слотов.

**Связи:** BR-002, NFR-004, UC-003 (6a)

## FR-010 · P1 · Отмена записи клиентом до дедлайна

**Описание:** Клиент может отменить свою предстоящую запись при соблюдении следующих правил:

- Если до начала тренировки **более 10 минут** — отмена разрешена **без штрафа**. Место освобождается.
- Если до начала тренировки **10 минут или менее** — отмена разрешена **со штрафом** (начисляется штрафной балл). Место освобождается.

**Штрафная механика:** при накоплении 3 штрафных баллов запись на новые тренировки блокируется на 24 часа. Штрафные баллы отображаются в профиле клиента.

**Очистка штрафных баллов:** Штрафные баллы **сбрасываются после успешного посещения 5 тренировок подряд без штрафов**. Успешным посещением считается тренировка, на которой клиент присутствовал (поле `attended = true` в данных брони).

**Отмена при блокировке:** Блокировка записи не влияет на возможность отмены существующих записей.

**Связи:** US-006, UC-005

## FR-011 · P1 · Поведение при отмене с начислением штрафа

**Описание:** При попытке отмены **менее чем за 10 минут** до начала приложение:

- отображает предупреждение: **«Вы отменяете запись за X минут до начала. Будет начислен штрафной балл. Продолжить?»**;
- запрашивает подтверждение;
- при подтверждении выполняет отмену через API и начисляет штрафной балл.

**Связи:** UC-005 (4a)

## FR-012 · P1 · Просмотр своих предстоящих записей

**Описание:** Клиент видит список своих активных (предстоящих) бронирований с деталями слота и статусом.

**Статусы брони:**

- `active` — активная (предстоящая)
- `cancelled_by_client` — отменена клиентом
- `cancelled_by_club` — отменена скалодромом
- `completed` — завершена (клиент присутствовал)
- `no_show` — неявка (клиент не пришёл)

**Связи:** US-002, UC-004

## FR-013 · P1 · Отображение цены после записи

**Описание:** После успешной записи и в карточке брони отображается **итоговая цена** тренировки. При выборе проката на экране подтверждения отображаются:

- стоимость тренировки без проката;
- стоимость проката (если API возвращает тариф отдельно);
- итоговая цена (сумма тренировки и проката).

**Ограничение:** статусы «ожидает оплаты / оплачено» **не требуются**; онлайн-оплата **не реализуется**.

**Связи:** BR-010, US-006

## FR-014 · P1 · Статус «Отменена скалодромом» с причиной

**Описание:** Если слот отменён организатором (профилактика, болезнь инструктора и т.д.), бронь клиента **не удаляется**, а переводится в статус **«Отменена скалодромом»** с текстом **причины**, полученной от API.

**Связи:** R-008, BR-009, US-007, UC-006

## FR-015 · P1 · Push-уведомление об отмене скалодромом

**Описание:** При смене статуса брони на «Отменена скалодромом» клиент получает **push-уведомление** с указанием причины. При следующем открытии приложения, если есть брони со статусом «Отменена скалодромом», показывается модальное окно с причиной отмены (fallback-механизм на случай недоставки push).

**Примечание:** для напоминаний согласован SMS; для отмены скалодромом — push (R-008).

**Связи:** BR-009, UC-006

## FR-019 · P2 · Оценка инструктора звёздами 1–5

**Описание:** После завершения тренировки клиент может поставить инструктору оценку **от 1 до 5 звёзд** в течение **24 часов** после окончания слота.

**Ограничения:**

- оценка доступна только для посещённых/завершённых тренировок (логика явки — на стороне бэкенда, поле `attended` в ответе API);
- в карточке слота до записи отображается **средний рейтинг инструктора** и **количество оценок**;
- push-уведомление с приглашением оценить инструктора содержит **глубокую ссылку** на экран оценки конкретной тренировки.

**После отправки оценки:** Кнопка оценки становится неактивной с текстом: **«Вы уже оценили эту тренировку»**.

**Связи:** BR-007, US-009, UC-008

## FR-020 · P2 · Программа лояльности в клиентском приложении

**Описание:** Постоянным клиентам в приложении отображается статус участия в программе лояльности и связанные преимущества.

**Механика:**

- Скидка **50% на каждое 10-е посещение**.
- В профиле клиента отображается **счётчик посещений** и прогресс: **«До скидки осталось X тренировок»**.
- Скидка применяется **автоматически** при записи на 10-ю тренировку. Цена пересчитывается до подтверждения записи.
- Скидка применяется **вместо** других действующих скидок (не суммируется).
- Клиент видит применённую скидку на экране подтверждения записи отдельной строкой.
- **После применения скидки счетчик продолжает накапливаться** (11-е посещение считается как 1-е к следующей скидке, 12-е как 2-е и т.д.).
- При ошибке API лояльности блок со скидкой скрывается, основная функциональность записи сохраняется.

**Связи:** BR-008, US-010, UC-009

## FR-027 · P2 · Управление сессией клиента

**Описание:**

- Сессия клиента сохраняется между запусками приложения (токен хранится в защищённом хранилище).
- При повторном открытии приложения выполняется автоматический вход по сохранённому токену.
- При невалидном токене клиент перенаправляется на экран входа.
- В профиле клиента доступна кнопка **«Выйти»**. При нажатии токен удаляется, клиент переходит на экран входа.
- Смена номера телефона доступна **только через обращение к администратору скалодрома** (не в приложении).

**Связи:** NFR-010, UC-001

## FR-028 · P2 · Отображение статуса штрафных баллов

**Описание:** В профиле клиента отображается:

- статус блокировки: **«Запись заблокирована на 24 часа. Осталось: X ч Y мин»** (при наличии 3 штрафных баллов);
- прогресс очистки штрафных баллов: **«До сброса штрафных баллов осталось N успешных посещений»**.

**Связи:** FR-010, US-006

## FR-029 · P2 · Настройка уведомлений

## FR-031 · P2 · История записей

## FR-032 · P2 · Обработка ошибок API по HTTP-статусам

**Описание:** Клиентское приложение корректно обрабатывает следующие HTTP-статусы, возвращаемые API:


| Статус | Описание                  | Действие приложения                                                          |
| ------ | ------------------------- | ---------------------------------------------------------------------------- |
| 400    | Невалидный запрос         | Показать сообщение: «Некорректные данные. Проверьте введенную информацию»    |
| 401    | Неавторизован             | Очистить сессию, перенаправить на экран входа                                |
| 403    | Доступ запрещен           | Показать сообщение: «У вас нет прав для выполнения этого действия»           |
| 404    | Не найдено                | Показать сообщение: «Запрашиваемая информация не найдена»                    |
| 409    | Конфликт                  | Показать сообщение: «Место уже занято. Попробуйте выбрать другую тренировку» |
| 422    | Невалидные данные         | Показать сообщение с деталями ошибки от API                                  |
| 429    | Too Many Requests         | Показать сообщение: «Слишком много попыток. Попробуйте через 15 минут»       |
| 500    | Внутренняя ошибка сервера | Показать сообщение: «Что-то пошло не так. Попробуйте позже»                  |
| 503    | Service Unavailable       | Показать сообщение: «Сервис временно недоступен. Попробуйте позже»           |
| 504    | Gateway Timeout           | Показать сообщение: «Сервер не отвечает. Попробуйте позже»                   |


**Связи:** NFR-004, UC-003

---
